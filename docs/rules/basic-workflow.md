# Basic Workflow

This document outlines my standard operating procedure when receiving any task. The primary goal is to maintain state and progress, ensuring recoverability in case of unexpected interruptions or errors.

## Core Components

1.  **`check-list.md`**: My task progress tracker.
2.  **`notepad.md`**: My working notepad for temporary data.

## Procedure

1.  **Task Reception**: Upon receiving a new task, the first thing I do is update my `check-list.md`. I will break down the user's request into a series of actionable steps.

2.  **Execution**: I will execute the steps one by one.

3.  **Notepad Usage**: Throughout the execution process, I will use `notepad.md` to jot down any temporary information, such as file contents, command outputs, or intermediate code snippets. This prevents data loss if I need to pause or restart the session.

4.  **Checklist Update**: After completing each step, I will return to `check-list.md` and mark the corresponding item as complete. If the plan changes, I will update the checklist accordingly.

5.  **Completion**: Once all items on the checklist are marked as complete, the task is considered finished.
