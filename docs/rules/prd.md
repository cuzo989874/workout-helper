# 產品需求文件 (Product Requirements Document) - 運動計時器

## 1. 專案概述與目標 (Project Overview & Goals)

### 專案概述
本專案旨在為現有的運動追蹤應用程式引入一個強大且直觀的「運動計時器」功能。此計時器將引導用戶完成其訓練計畫中的每個運動組數和休息時間，確保訓練過程的結構化和效率。

### 專案目標
*   提供一個無縫且有引導性的訓練體驗。
*   透過自動化計時，幫助用戶在訓練期間保持專注。
*   確保運動組數和休息時間的精確追蹤。
*   提升用戶對應用程式的參與度和滿意度。

## 2. 用戶畫像 (User Personas)

### Persona 1: 「專注的 David」
*   **描述:** David 是一位定期健身的運動愛好者，他遵循嚴格的訓練計畫。他需要精確的運動組數和休息時間計時，以最大化訓練效果。如果應用程式無法跟上他的節奏或需要手動計時，他會感到沮喪。
*   **需求:** 精確的倒數計時、清晰的視覺/聽覺/觸覺提示、能夠快速在組數/休息之間切換、最少的干擾。

### Persona 2: 「休閒的 Carol」
*   **描述:** Carol 剛開始健身，有時會忘記計時休息時間，或者不確定每個運動應該持續多久。她需要一個簡單、易於遵循的引導，幫助她保持訓練進度，而不會感到不知所措。
*   **需求:** 清晰的指示、針對無計時組數的簡單「下一步」選項、自動進程、易於理解的介面。

## 3. 功能列表與優先級 (Feature List with Priorities)

## 4. 詳細用戶故事與驗收條件 (Detailed User Stories and Acceptance Criteria)

#### 1. 啟動計時器

*   **US-WT-001: 從運動詳情頁啟動計時器**
    *   **作為一個運動者，** 我希望在「Workout Detail (運動詳情)」頁面看到一個「Start Workout (開始訓練)」按鈕，**以便我能啟動當前訓練的計時模式。**
    *   **驗收條件:**
        *   「Start Workout」按鈕應作為右上角的行動按鈕之一顯示。
        *   如果訓練計畫中沒有任何運動 (Exercise) 或組數 (Set)，則不顯示此按鈕。

#### 2. 運動組數 (Set) 計時與流程

*   **US-WT-002: 執行有計時的運動組數**
    *   **作為一個運動者，** 當運動組數 (Set) 設定了計時時間時，我希望計時器能顯示倒數計時，**以便我能知道該組運動需要持續多久。**
    *   **驗收條件:**
        *   計時器應以全螢幕顯示，並包含一個圓形進度條。
        *   計時器中間顯示碼錶，格式為 `mm:ss`；若剩餘時間小於一分鐘，則顯示 `ss`。
        *   倒數至最後 10 秒時，計時器顏色應轉換以提示即將結束。
        *   計時結束後，系統應自動進入下一步（休息模式或下一個組數/運動）。
        *   用戶在計時過程中，應能手動點擊「下一步」按鈕提前進入下一階段。

*   **US-WT-003: 執行無計時的運動組數**
    *   **作為一個運動者，** 當運動組數 (Set) 沒有設定計時時間時，我希望看到一個「下一步」按鈕，**以便我可以在完成該組運動後手動進入休息模式。**
    *   **驗收條件:**
        *   按鈕文案應顯示為「下一步」。
        *   用戶長時間不點擊「下一步」按鈕時，系統不應有任何自動行為或提示。

#### 3. 休息時間 (Rest Time) 處理

*   **US-WT-004: 執行有計時的休息時間**
    *   **作為一個運動者，** 當運動組數完成後，如果設定了休息時間，我希望計時器能顯示休息時間的倒數計時，**以便我能充分休息並準備下一組運動。**
    *   **驗收條件:**
        *   休息計時器應以大字樣顯示休息時間，並同時顯示下一組的內容。
        *   休息倒數計時結束時，系統應自動進入下一組運動。
        *   用戶在休息計時過程中，應能手動點擊「下一步」按鈕提前跳過休息，進入下一組。

*   **US-WT-005: 自動跳過無計時的休息**
    *   **作為一個運動者，** 當運動組數完成後，如果沒有設定休息時間，我希望系統能自動跳過休息，直接進入下一個運動組數或運動，**以便我的訓練流程更加流暢。**
    *   **驗收條件:**
        *   在自動跳過休息時，UI 上不需要有任何特別的提示。

#### 4. 訓練流程控制與完成

*   **US-WT-006: 訓練流程導航**
    *   **作為一個運動者，** 我希望在計時器介面中能清晰地看到當前正在進行的運動名稱和組數，並顯示總進度（例如：X/Y 組），**以便我能隨時掌握訓練進度。**
    *   **驗收條件:**
        *   計時器介面應清晰顯示當前運動名稱、當前組數，以及總進度（例如：「第 2 組 / 共 4 組」）。
        *   僅提供「下一步」按鈕，不提供「上一個組數」或「跳過運動」等導航功能。

*   **US-WT-007: 訓練完成提示**
    *   **作為一個運動者，** 當所有運動和組數都完成時，我希望看到一個「訓練完成」的提示畫面，**以便我能知道本次訓練已結束。**
    *   **驗收條件:**
        *   「訓練完成」畫面只需顯示「訓練完成」的訊息。
        *   完成後，用戶點擊按鈕應返回主頁。

#### 5. 計時器通用功能

*   **US-WT-008: 暫停與恢復計時**
    *   **作為一個運動者，** 我希望在計時器運行過程中能夠暫停和恢復計時，**以便我能應對突發情況或短暫休息。**
    *   **驗收條件:**
        *   暫停和恢復功能應透過 icon-btn 呈現。
        *   暫停時，計時器應停在用戶按下暫停時的秒數。
        *   用戶可以選擇按下「恢復」繼續計時，或按下「下一步」直接跳過當前階段，開始下一步的計時。

*   **US-WT-009: 異常處理**
    *   **作為一個運動者，** 如果訓練計畫中存在沒有組數的運動，或者組數的計時時間或休息時間設定為 0，我希望系統能有合理的處理方式，**而不是導致應用程式崩潰或卡住。**
    *   **驗收條件:**
        *   如果 `set.time` 設定為 0，應視為 `null` 或 `undefined`，讓用戶手動點擊「下一步」結束當前動作。
        *   如果 `rest.time` 設定為 0，應直接跳過休息時間，即沒有休息時間。
        *   如果一個運動 (Exercise) 沒有設計任何組數 (Set)，應視為有 1 組。
