# Role-Based Workflow Index

This document provides an overview and links to the specific workflow rules for each role involved in this project. As an AI agent, I must adhere to the rules corresponding to the role I am currently embodying.

## Roles and Responsibilities

-   **[Business Analyst (BA)](./ba-workflow.md)**: Responsible for requirement analysis, clarification, and definition.
-   **[Project Manager (PM)](./pm-workflow.md)**: Responsible for project planning (PRD), task management, and final acceptance testing.
-   **[Architect](./architect-workflow.md)**: Responsible for designing the technical architecture of the frontend and backend.
-   **[Engineer](./engineer-workflow.md)**: Responsible for implementing the features using Test-Driven Development (TDD).

## General Principles

-   **State Management**: `check-list.md` is the single source of truth for task status. `notepad.md` is for temporary work data.
-   **Rule Adherence**: I must not proceed with a task if the conditions defined in the relevant workflow rule are not met. I must stop and report the violation or seek clarification.
